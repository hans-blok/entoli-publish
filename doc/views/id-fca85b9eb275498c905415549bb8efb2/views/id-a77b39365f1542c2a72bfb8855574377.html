DESCRIPTION This diagram is an abstract model of the business objects within the Entoli ecosystem. It shows four interrelated areas: Project, Entoli ecosystem, Model (with Semantic Foundation) and Language Intelligence Resource as external context, with their mutual relationships. OUR APPROACH We separate client environments. Each client has its own configuration for using an LLM (e.g. API keys and the choice of which model is used). This can be a client's own model, an external model such as Anthropic, OpenAI or Google, or a model that Entoli itself offers, for example Ollama. Every Agent acts on the basis of rules. These rules exclusively use concepts that are recorded in the semantic model; other concepts are not permitted. Here we apply graph theory: a concept is either an element or a relationship. This is advisable because it keeps the semantic model the single source of truth for what an agent may reason about and execute. By limiting concepts to a closed, formalised model, you prevent agents from introducing deviating or inconsistent notions, rules remain consistent and reusable across agents, and an agent's behaviour can be traced back to and checked against the model. The graph-theoretic division into elements and relationships also aligns with standards such as ArchiMate, which makes the model exchangeable and extensible. WHY IS A GRAPH USEFUL? Think of an LLM as a very well-trained gambler: based on the text so far, it keeps predicting the most probable next word. There is no database inside it with fixed facts or rules — all "knowledge" is woven into billions of numbers (weights) learned during training. That makes an LLM strong at language and flexible reasoning, but also unreliable: it can make up things that sound plausible but are not correct (hallucinate), and it does not inherently know which concepts do or do not exist within your domain. A graph solves exactly that. A graph consists of nodes (elements) and connections (relationships), like the semantic model above. By not leaving that knowledge only implicit in the language model, but explicitly recording it as a graph, you give the LLM two things it otherwise lacks: - Grounding: you supply the LLM, prior to its reasoning, with the relevant nodes and relationships from the graph, instead of the model having to guess what exists and how it relates. - Boundary-setting: by stating "a concept is always an element or a relationship from this semantic model", the model cannot invent its own, undefined concepts. The graph thus acts as a fence around the reasoning. In short: the LLM is good at language, the graph is good at consistency and checkability. By combining them, an Entoli agent reasons within the lines of the semantic model instead of free-associating.
