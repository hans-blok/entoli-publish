Synthesis-regime ()
Synthesis Regime is the data object that records, per agent-intent, which meaning-transformations the LLM is permitted to perform once its sources are loaded and its task shape is fixed. It carries exactly one enumerated value per agent-intent, drawn from a fixed set of three. It answers a question distinct from the two objects it is associated with: Source Regime records which sources may be loaded; Interpretation Mode records what shape of task this is; Synthesis Regime records what may be done to meaning once both of those are settled. Attributes - value (Enumeration): One of: Faithful, Structuring, Generative - scope (Reference): The agent-intent this value applies to - epistemicGuarantee (Derived boolean): true for Faithful/Structuring, false for Generative unless the generated content has since been validated and canonised Enumeration values - Preserving: no new meaning introduced; rephrasing, summarising, and strict derivation from source only. - Relating: meaning unaltered; only reorganisation, relation-making, or structural exposure of existing content permitted. - Generating: new meaning (hypotheses, assumptions, alternative framings) may be introduced; every such instance must be explicitly marked and carries no epistemic guarantee until validated and canonised. Relationships - Association -> Agent Contract: every agent-intent within a contract carries exactly one Synthesis Regime value. - Association (orthogonal) -> Source Regime: independent axes; an Open source regime does not by itself grant Generative permission. - Association (orthogonal) -> Interpretation Mode: independent axes; the same task shape may be declared under any Synthesis Regime value. - Realization -> Epistemic Guarantee: Faithful and Structuring preserve it on every claim; Generative suspends it for newly introduced content. Constraints - MUST be declared explicitly per agent-intent; never inferred from Source Regime or Interpretation Mode. - Extracted from language previously embedded in Source Regime's Open position and in Interpretation Mode's per-position constraints - a relocation of existing behaviour, not a new permission.
