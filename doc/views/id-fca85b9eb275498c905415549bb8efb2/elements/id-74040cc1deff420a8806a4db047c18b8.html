Task Mode ()
Task Mode is the data object that records, per agent-intent, which type of operation is to be executed and what output structure must be produced. It carries exactly one enumerated value per agent-intent, drawn from a fixed set of five. It answers a question distinct from the other objects it is associated with: Source Regime records which sources may be loaded; Synthesis Regime records what may be done to meaning; Execution Strategy records how execution progresses; Task Mode records what kind of operation is performed and how the result must be structured. Enumeration values - Extracting: isolate explicitly present elements from source material; output is a list or set of elements; no additional relationships introduced; all elements MUST be directly traceable to source. - Structuring: organise and relate existing elements without altering meaning; output is a structured artifact (e.g. hierarchy, relations, sections); no new meaning or relationships beyond source. - Transforming: reformulate content into a different representation while preserving meaning; output is an equivalent artifact in a new form; semantic equivalence MUST be maintained. - Evaluating: assess content against explicitly defined criteria; output consists of criteria, per-criterion assessment, and explicit conclusion; criteria MUST be stated and traceable. - Generating: produce new content not fully present in the source; output is a coherent artifact fulfilling the task goal; new meaning is allowed only as governed by Synthesis Regime. Constraints - MUST be declared explicitly per agent-intent; never inferred from Source Regime, Synthesis Regime, or Execution Strategy. - MUST define: - the type of operation performed - the structure of the output produced - MUST NOT define: - iteration or looping - branching or step selection - progression logic - Extracted from language previously embedded in Interpretation Mode and task descriptions - a relocation and formalisation of existing behaviour, not a new capability.
