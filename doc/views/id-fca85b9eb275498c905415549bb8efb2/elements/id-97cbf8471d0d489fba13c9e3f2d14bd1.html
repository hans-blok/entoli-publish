# Realisation **Realisation** is the phase in which established meaning is converted into a working, applicable or executable structure within the workspace. Where **Recording** makes meaning binding, **Realisation** makes that meaning operational. Realisation produces realising artefacts. ## Characteristics - Converts bindingly established meaning into operation - Creates executable, applicable or working structures - Reduces interpretive latitude - Increases concreteness and specificity - Shifts focus from "what must hold" to "what works" Interpretation Mode: Realising --- ## Examples (value-stream generic) - Implementing software code - Setting up a workflow or process - Configuring a runtime environment - Building a data model in a database - Implementing a governance mechanism - Setting up an organisational structure - Realising an investment structure [The original Dutch text embedded a duplicate "4.4 Operationalisatie" sub-section here, describing content that belongs to the separate "6.
