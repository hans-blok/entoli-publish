# Representation within the Workspace A **representation** is the form in which meaning is recorded, displayed, or exchanged. A representation describes *how* something is expressed — not *what* it means and not *what role* it fulfils in the workspace. It is independent of: - the process phase in which something arises (development phase), - the nature of the semantic intervention (intervention type), - the governance role of the artifact (artifact-function-axis). A representation is an **expression form of meaning**. --- ## Definition A representation is a concrete expression of an artifact in a specific form, suitable for human or machine processing. The same meaning can exist in multiple representations without the underlying meaning changing. --- ## Characteristics - Form-bound, not meaning-determining - Can be translated to other representations - Has no normative force of its own - Can be leading or derived (see Derivation Position) - Serves a specific purpose (readability, executability, visualization, processing) --- ## Typical representation forms - **Text**: Markdown, Plain text, Gherkin, Guideline document - **Diagram**: ArchiMate view, UML diagram, Process diagram, Capability map - **Model data / Code**: JSON, YAML, XML, SQL, Programming language - **Machine representation**: Compiled code, Binary artifacts, Runtime configuration --- ## Examples | Meaning | Possible representations | |---------|--------------------------| | API contract | OpenAPI (YAML), generated HTML documentation | | Domain model | UML diagram, JSON model, database DDL | | Architecture principle | Markdown document, Wiki page, PDF | | Requirements | Gherkin file, generated test report | In all cases the meaning remains the same while the form differs. --- ## Relation to other ordering concepts - **Artifact-function-axis** determines what the artifact means in the workspace (normative, structuring, realising, descriptive). - **Intervention Type** describes what a role does with meaning. - **Development phase** describes where in the chain it arises. - **Representation** describes exclusively the form in which it exists.
