Execution strategy ()
Execution Strategy is the data object that records, per agent-contract, how Task Modes are selected, sequenced, and iterated during execution. It carries exactly one enumerated value per agent-contract, drawn from a fixed set of three. It answers a question distinct from the other objects it is associated with: Source Regime records which sources may be loaded; Synthesis Regime records what may be done to meaning; Task Mode records what type of operation is performed; Execution Strategy records how execution progresses across those operations. Attributes - value (Enumeration): One of: Linear, Exploratory, Convergent - scope (Reference): The agent-contract this value applies to - progressionDeterminism (Derived enumeration): deterministic for Linear, adaptive for Exploratory, selective for Convergent Enumeration values - Linear: execution follows a predefined sequence of Task Modes; no iteration or branching; progression is fully deterministic and terminates upon completion of the sequence. - Exploratory: execution proceeds iteratively; Task Modes are selected based on intermediate results; progression is adaptive and continues until sufficient understanding or stability is reached. - Convergent: execution begins with multiple candidates (options, hypotheses, alternatives) and reduces them over successive iterations; progression is driven by evaluation and elimination toward a single validated outcome. Constraints - MUST be declared explicitly per agent-contract; never inferred from Task Mode, Source Regime, or Synthesis Regime. - MUST be realised explicitly in orchestration; execution flows must reflect the declared strategy. - MUST NOT define: - allowed sources - allowed meaning transformations - structure of individual tasks - Extracted from previously implicit sequencing, iteration, and decision logic embedded in orchestration flows - a relocation and formalisation of existing behaviour, not a new capability.
